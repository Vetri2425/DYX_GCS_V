## USER WORKFLOW

1. APP START
   ├─ Check AsyncStorage for saved preference
   ├─ If found: Use saved state
   └─ If not: Fetch from /api/tts/status

2. USER TAPS BUTTON
   ├─ Show loading spinner
   ├─ POST to /api/tts/control with new state
   │
   ├─ ON SUCCESS:
   │  ├─ Update button state
   │  ├─ Save preference to AsyncStorage
   │  └─ Show success alert
   │
   └─ ON ERROR:
      ├─ Show error alert
      └─ Revert to previous state

3. PERSISTENCE
   └─ User preference saved automatically and restored on app restart
